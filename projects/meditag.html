회원이 약 복용 알림을 등록하고, 약사가 안내하는 주의사항을 음성으로 녹음해 저장할 수 있는 기능을 구현

시각장애인이 약 복용 중 주의해야 할 내용을 필요할 때마다 반복해서 들을 수 있도록 구성

SMS 복용 알림 기능 구현

회원이 약을 복용 완료로 변경하면, 보호자에게 해당 정보를 실시간으로 전달하는 기능을 구현했습니다.

보호자의 전화번호·회원 이름·약 이름을 기반으로 CoolSMS API를 호출해 문자 알림을 전송하도록 구성했으며, 이를 통해 보호자가 복용 현황을 즉시 확인하고 모니터링할 수 있도록 지원했습니다.

트러블슈팅

S3 Presigned URL 기반 음성·이미지 업로드 구조 개선
문제점
  • 기존 업로드 방식: 클라이언트 → 서버 → S3
  • 파일 크기에 따라 서버가 동일 파일을 두 번 처리해야 해 네트워크 I/O가 2배로 증가
  • 대용량 파일 업로드 시 서버의 메모리·트래픽·CPU 사용량 증가로 성능 저하 우려
해결
  • 클라이언트가 S3에 직접 업로드하도록 Presigned URL 방식 적용
  • 서버는 파일 자체를 처리하지 않고 메타데이터만 관리하도록 구조 개선
결과
  • 서버의 파일 처리 의존도 0%로 전환
  • 네트워크 I/O 기존 대비 약 50% 감소
  • 업로드 성능 안정화 및 서버 부하 대폭 감소
만료된 알림 지속 발송 문제
🖼 알림 기능 화면 이미지 추가예정

시각장애인이 복용 시간을 놓치지 않도록, 사용자가 등록한 복용 시간에 맞춰 알림을 제공하는 기능을 구현했습니다. Spring 스케줄러를 활용해 약 복용 시간을 기준으로 알림을 자동 예약·발송하도록 설계하여, 사용자가 직접 시간을 확인하지 않아도 복약 리마인드를 받을 수 있도록 했습니다.

문제점
  • 복용 종료일 이후에도 알림이 계속 발송되는 오류 발생
  • 종료된 복약에 대한 불필요한 알림으로 사용자 혼란 발생
해결
  • 알림 시간이 현재 시각보다 이전이거나 유효 기간(endDate)을 벗어난 경우 스케줄링하지 않도록 검증 로직 추가
  • 복용 종료일 다음 날 모든 알림을 자동 취소하는 기능 구현
결과
  • 만료된 복약 알림의 지속 발송 문제 완전 제거
  • 알림 신뢰도 및 사용자 경험 개선
서버 시간대 문제로 인한 알림 지연
문제점
  • EC2 서버 기본 시간대가 서울 기준이 아니어서 알림이 최대 9시간 지연 발송되는 문제 발생
해결
  • 서버 타임존을 Asia/Seoul 기준으로 재설정
결과
  • 사용자 설정 시간에 맞춰 알림이 9시간 지연 → 0초 정확히 발송되도록 정상화
  • 시간 기반 기능의 신뢰성 확보
