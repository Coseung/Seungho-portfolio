문제점 & 해결

문제점
  • 스레드 사용자들은 팔로워를 늘리기 위해 하루 수십차례 스하리(좋아요, 리포스트, 댓글, 팔로우)를 반복
  • 스레드 사용자들은 스하리를 하여 댓글을 달면 상대방이 반대로 팔로우를 해주는 룰이 있음
  • 스하리를 최대한 많이 하여 돌아오는 팔로우로 팔로워를 모음
요구사항
  • 반복되는 스하리를 클릭 한번으로 자동화
  • 실제 사용자 피드백 반영 (댓글 기능 추가, 스크롤 딜레이 적용 요구)
해결
  • 웹 브라우저에서 자동으로 테스트를 실행할 수 있게 도와주는 도구 Python Selenium을 선택
  • 다수 사용자가 손쉽게 실행할 수 있도록 Windows 실행 파일(.exe)로 패키징하여 별도 개발 환경 없이 실행 가능하게 구성
  • 사용자 피드백 반영하여 댓글 기능 및 스크롤 딜레이 기능 추가 개선
  • 개선 후 활성 사용자 12명 → 25명으로 약 2배 증가 (전체 사용자 32명)

프로그램 기능

무료 버전

유료 버전

무료 버전 기능
  • 고정 키워드 "스하리 1000" 게시물 자동 찾기
  • 게시물 추출, 자동 처리
  • 자동 팔로우 기능
  • 자동 좋아요 기능
  • 자동 리포스트 기능
  • 사용자 지정 댓글 자동 작성 (한줄 텍스트만 가능 이모티콘 불가)
  • 실행당 스하리 수: 15~35개 제한
  • 스크롤 횟수: 6번으로 제한
유료 버전 기능
  • 무료 버전 기능 포함
  • 키워드 직접 작성
  • 스크롤 횟수 조정 가능 (무료버전은 6번으로 제한)
  • 실행당 스하리 수: 120개 이상 가능
  • 딜레이 시간 조정 가능 (사람처럼 행동, 계정 안전 목적)
  • 댓글 txt 파일 첨부 기능
  • 입력, 첨부한 댓글 중 랜덤으로 작성

무료/유료 버전은 설정 옵션의 범위(키워드, 스크롤, 딜레이, 처리량, 댓글 방식)를 달리해, 무료는 체험용으로 핵심 기능을 제한 제공하고 유료는 대량 실행과 계정 안전을 고려한 고급 설정을 제공하도록 구성했다.

수익화 단계

접근 방식
  • 초기 진입장벽을 낮추기 위해 '바로 판매'가 아니라 무료 배포 → 유료 전환 흐름으로 설계
  • 무료는 핵심 기능을 체험할 수 있게 제공하되 처리량·설정 옵션을 제한하여, 사용량이 늘수록 유료 버전이 필요해지도록 구성
유입 방법
  • 스레드(Threads)에 관심을 끌 수 있는 안내 글을 게시하고, 댓글 및 SNS 메시지로 무료 사용법 Notion 페이지 링크를 전달
전환 유도
  • 무료 사용법 Notion 페이지에 유료 버전 기능과 차이점을 명확히 안내하여 업그레이드 문의로 이어지도록 구성
판매 진행
  • 구매 문의 및 거래는 오픈채팅으로 연결해 진행
요약 프로세스
  • Threads 게시물 게시 → 댓글/DM 응대 → Notion 가이드 전달 → 유료 기능 안내 → 오픈채팅 거래
