02 · 담당 기능 및 구현 범위

담당 기능 및 구현 범위

직원(임직원/크리에이터) 관리 CRUD
  • 직원 등록·조회·수정·해고 처리 RESTful API 설계 및 구현
  • 관리자 전용 전체 직원 조회, 매니저/크리에이터 직급별 필터링 기능 구현
  • AWS S3 연동 프로필 이미지 업로드 및 관리 기능 구현
일정 관리 CRUD
  • 전사 공통 일정과 개인 일정을 분리 관리하는 이중 스케줄 구조 설계
  • 매니저가 담당 크리에이터의 일정을 대신 등록, 수정, 삭제할 수 있는 위임 일정 관리 로직 구현
  • 일정 유형(미팅, 콘텐츠, 프로모션, 개인 등) 분류 기반 필터링 API 구현
WebSocket + STOMP 실시간 채팅
  • STOMP 프로토콜 기반 단체 채팅방 및 1:1 채팅 기능 구현
  • 채팅방 입장·퇴장 이벤트 처리 및 브로드캐스트 구독 관리 구현
  • 메시지 수신 시 읽음 처리 로직 구현 (읽지 않은 메시지 카운트 관리)
  • clientUuid 식별자 도입으로 브로드캐스트 중복 렌더링 방지
  • 마지막 메시지 ID 기준 커서 페이징으로 응답 시간 70ms → 14ms, 약 80% 단축
전역 예외 처리 시스템 구축
  • @RestControllerAdvice 기반 글로벌 예외 핸들러 구현으로 API 전반 에러 응답 포맷 통일
  • CustomException + ErrorCode Enum 설계로 도메인별 비즈니스 예외를 명시적으로 정의
03 · 트러블슈팅 — 직원관리 & 예외처리

트러블슈팅

직원관리 및 전역 예외처리
문제점
  • 직원 목록 통계를 Attendance, Vacation, MemberEmployeeDetail 테이블에서 각각 쿼리하여 단일 API 호출에 다수의 DB 요청 발생
  • 직원 등록 시 세 테이블에 순차 저장하는 구조에서 중간 실패 시 일부 데이터만 저장되는 문제 발생
해결
  • Fetch Join으로 연관 테이블을 한 번에 조회하여 단일 API 호출로 목록과 통계를 모두 응답
  • 등록 전체 로직을 @Transactional로 묶어 중간 실패 시 전체 롤백 처리
결과
  • 단일 API 호출로 직원 목록과 4가지 통계를 한 번에 응답하는 구조로 개선
  • 연관 테이블 저장 중 일부만 저장되는 문제 차단

ErrorCode Enum에 HTTP 상태 코드와 에러 메시지를 함께 정의하고 CustomException이 이를 참조하는 구조로 설계. @RestControllerAdvice 기반 전역 예외 핸들러로 CustomException 및 일반 예외를 일괄 처리하여 에러 응답 포맷 통일

04 · 트러블슈팅 — WebSocket 읽음처리

트러블슈팅

WebSocket + STOMP — 읽음 처리 DB 설계 개선
문제점
  • 초기 설계: ChatMessage 테이블에 read 컬럼으로 메시지마다 읽음 여부를 개별 관리
  • 단체 채팅방에서 메시지 수신 시 N개의 레코드 각각에 UPDATE 쿼리 발생 — 메시지 양에 비례한 쿼리 증가 우려
해결
  • ChatRoomMember 테이블에 lastReadMessageId 컬럼 추가 방식으로 설계 전환
  • 메시지 전체를 수정하는 대신 사용자별 마지막 읽은 메시지 ID 1건만 UPDATE
  • SimpUserRegistry로 현재 구독 중인 접속자를 실시간 조회, 접속자는 즉시 / 비접속자는 진입 시 일괄 처리
결과
  • 읽음 처리 UPDATE 쿼리: 기존 메시지 N건 UPDATE → 멤버 1건 UPDATE로 개선
  • STOMP 브로드캐스트 중복 렌더링 문제는 clientUuid 식별자 도입으로 필터링하여 해결
05 · 트러블슈팅 — 채팅 메시지 페이징

트러블슈팅

채팅 메시지 조회 DB 설계 개선
문제점
  • 채팅방 입장 시 전체 메시지를 한 번에 조회하여 메시지 2,000건 기준 평균 70ms, 최대 198ms의 응답 지연 발생
해결
  • 오프셋(Offset) 방식 대신 마지막 메시지 ID를 기준으로 조회하는 커서 기반 페이징 적용
  • 채팅방 입장 시 최신 메시지 20건만 우선 조회, 사용자가 위로 스크롤할 때만 이전 메시지 20건 추가 로드
결과
  • 평균 응답 시간 70ms → 14ms 로 약 80% 감소
  • 최대 응답 시간 198ms → 10ms 로 약 95% 감소
